Details

The multiplicity of a storage index is the number of tensor components it represents. For example, if a tensor is symmetric then the multiplicity for the storage index's representing symmetric terms have a multiplicity of 2. ◆ increment_tensor_index

template<typename IndexList , typename TensorIndex >
using TensorMetafunctions::increment_tensor_index = typedef typename detail::increment_tensor_index_impl<( tmpl::at<TensorIndex, tmpl::size_t<0> >::value + 1 >= tmpl::at<IndexList, tmpl::size_t<0> >::dim)>::template type<IndexList, TensorIndex, 0>

Increment a tensor index.

Details

Given a tensor index TensorIndex and a list of TensorIndex's, IndexList, increment the tensor index to the next component. Incrementing is done by incremeting the first (left-most) index first, once it reaches the dimensionality of that index, the next index is increment. That is, for a tensor \(T_{abc}\) where all indices are 3-dimensional, incrementing the index \((1, 3, 0)\) results in the index \((2, 3, 0)\), and incrementing the index \((3, 3, 0)\) results \((0, 0, 1)\).

For a typelist of TensorIndexType's IndexList and a tensor index (typelist of std::integral_constant) TensorIndex,

using result = IncrementTensorIndex<IndexList, TensorIndex>;

Returns: typelist<std::integral_constant...>

Semantics: Let IndexList by a typelist

typelist<SpacetimeTensorIndex<3, UpLo::Lo, Frame::Grid>,
SpacetimeTensorIndex<3, UpLo::Lo, Frame::Grid>,
SpacetimeTensorIndex<3, UpLo::Lo, Frame::Grid>>;

and TensorIndex be tmpl::integral_list<int, 3, 3, 0>, then

using result = tmpl::integral_list<int, 0, 0, 1>;

◆ Symmetry

template<std::int32_t... T>
using Symmetry = typename detail::symm<T...>::type

Computes the canonical symmetry from the integers T

Details

Compute the canonical symmetry typelist given a set of integers, T. The resulting typelist is in descending order of the absolute value of the integers. For example, the result of Symmetry<1, 2, 3> is integral_list<int32_t, 3, 2, 1>. Anti-symmetries can be denoted with a minus sign on either or both indices. That is, Symmetry<-1, 2, 1> is anti-symmetric in the first and last index and is the same as Symmetry<-1, 2, -1>. Note: two minus signs are still anti-symmetric because it simplifies the algorithm used to compute the canonical form of the symmetry.

Template Parameters
Tthe integers denoting the symmetry of the Tensor
